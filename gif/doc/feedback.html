Feedback 

If you have any suggestions, comments, improvement ideas or new features you would like to see added to this component, please e-mail them to me: anders@melander.dk.

  Update notification

If you would like to be notified via e-mail when TGIFImage is updated, you can register with my mailing lists.
You can also register using the installation program. Just run the installation program, deselect all the components and fill in your email address when asked for it.

  Bug reports

Bug reports can either be reported on the Bug report form at my home page or mailed to the above address. If you mail bug reports to me, please provide the following information:

     • Your TGIFImage version number.
     • Your Delphi or C++ Builder version number (including updates).
     • Your Windows version (including service packs).
     • The color depth of your display adapter (# of color or bits/pixel).
     • The number of your secret swiss bank account (or your sister's phone number).

If possible, you should attach a ZIP file containing the smallest possible application that reconstructs the problem (don't include dsk, dsm or dcu files).
If you need to send me a screen shot, please send it in GIF format or as ZIPped BMP file.
If you are having problems displaying or loading a GIF file, please attach the problematic GIF file as well.

  Field tests

TGIFImage usually goes into field test 2-3 months before it is released. If you wish to participate in these field tests, send me an email stating so, and you will be notified when a field test is available.

I will need the following information from you:

     • How you are using TGIFImage.
     • The version(s) of Delphi and/or C++ Builder (including updates) you will be using TGIFImage with.
     • The Windows version(s) (including service packs) of the run-time system(s).
     • The color depth(s) of the run-time system(s).
